🔀 这次重构改了什么

❌ 旧（订单驱动）

  • 邮局每行 = 一张 post_office 订单
  • 邮局记录混进「订单列表 / 客户管理」——和电商真实订单重复、双算
  • 没有地方单独看邮局完整名册
  • 工单叫“挂订单”、改址叫“回流”（不好懂）

✅ 新（投递记录层）

  • 邮局每行 = 一条 投递记录（与中通同级）
  • 邮局记录不进「订单列表 / 客户管理」——营收/客户不再被污染
  • 新增 「投递名册」页签——完整名册的家（可搜可筛）
  • 工单叫 已关联读者 / 未匹配；改址叫 应用新地址
✅ 基本不变（沿用现有观感）：月度明细主从布局、各列、导入「预览→提交」流程、投递单位“有则标注无则留空”、明细冻结存档、收款发票整页。变的是底层实体（记录 vs 订单）和大白话叫法，不是推倒重做。
