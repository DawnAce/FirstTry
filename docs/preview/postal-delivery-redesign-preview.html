设计说明 / 待你拍板：
① 保留 emoji 页签（📇📦📮✏️📞🧾），你喜欢这个小图标——只是把标题栏收拾干净，选中项蓝色下划线，页签后带条数小徽标（如 投诉 6）。
② 砍掉横向滚动（每个页签都有这毛病）：这里以「投递名册」示范——常驻列 编号 / 收报人 / 地址·电话 / 份数 / 起止月 / 投递单位 / 渠道 / 操作，其余（邮编·产品·金额·业务员·汇款名·平台订单号·起止投日期）收进行内展开（示例第一行已展开）。
③ 顶部不做 4 个大数据框（按你说的）；只保留一条很轻的「概览行」：共 N 条 · 合计 M 份 · N 家投递单位 · X 条未填单位（未填是要补的活）。这条你要是嫌多，也能整条去掉。
④ 每个页签沿用同一套（干净工具栏 + 精简表格 + 行内展开），各自保留原有功能：
📦 月度起投明细保持「左月份列表 + 右名册」主从结构（本就不错）；顶部指标条 收件人/总份数/投递单位 保留；生成当月明细 / 标记已发·冻结 / 导出 照旧。
📮 投诉工单状态做成快筛（待处理/处理中/已解决 带计数）；处理时间线抽屉照旧；表格精简+展开。
✏️ 改地址突出「待应用」工作队列（已关联未应用 = 能干的活）；「应用新地址」按钮保留；新旧姓名地址对比展开。
📞 回访最轻，精简表格+展开长结果即可。
🧾 收款发票可选「概览行」：合计金额/到款 · 未挂单数；挂单标记保留；发票明细展开。
