# 操作手册

## 1. 首次使用

### 1.1 环境准备

#### 安装依赖软件
- **Python 3.11+**：从 [python.org](https://www.python.org/downloads/) 下载安装
- **Node.js 18+**：从 [nodejs.org](https://nodejs.org/) 下载安装
- **MySQL 数据库**：确保 MySQL 服务正在运行

#### 验证安装
```bash
# 检查 Python 版本
python --version

# 检查 Node.js 版本
node --version

# 检查 MySQL 是否运行
mysql --version
```

### 1.2 配置数据库连接

在项目根目录创建 `.env` 文件：

```env
MYSQL_HOST=localhost
MYSQL_PORT=3306
MYSQL_USER=your_username
MYSQL_PASSWORD=your_password
MYSQL_DATABASE=newspaper_system
```

**注意**：
- 确保数据库用户有创建表的权限
- 数据库 `newspaper_system` 需要提前创建好

### 1.3 启动后端服务

```bash
# 进入后端目录
cd backend

# 创建虚拟环境（仅首次）
python -m venv venv

# 激活虚拟环境
venv\Scripts\activate  # Windows
# 或
source venv/bin/activate  # macOS/Linux

# 安装依赖
pip install -r requirements.txt

# 运行数据库迁移
alembic upgrade head

# 启动服务
uvicorn app.main:app --reload --port 8000
```

**成功标志**：
```
INFO:     Uvicorn running on http://127.0.0.1:8000 (Press CTRL+C to quit)
```

### 1.4 初始化种子数据

打开新的命令行窗口（保持后端服务运行），执行：

```bash
curl -X POST http://localhost:8000/api/admin/seed
```

**响应示例**：
```json
{
  "message": "Seeded 52 schedule entries, 30 report templates"
}
```

**种子数据包含**：
- 2026 年刊期表（52 周，49 期出版）
- 报数模板配置（固定项 + 变动项）

### 1.5 启动前端服务（开发模式）

打开新的命令行窗口，执行：

```bash
# 进入前端目录
cd frontend

# 安装依赖（仅首次）
npm install

# 启动开发服务器
npm run dev
```

**成功标志**：
```
VITE ready in 500 ms

➜  Local:   http://localhost:5173/
```

### 1.6 访问系统

在浏览器中打开 `http://localhost:5173`，系统会自动跳转到登录页面。

**登录**：
- 输入用户名和密码（默认管理员账户：用户名 `admin`，密码 `admin123`）
- 点击"登录"按钮
- 登录成功后自动跳转到系统首页

### 1.6.1 顶栏全局搜索

登录后，**每个页面顶栏都有一个全局搜索框**，可跨模块快速检索并一键跳转：

- **能搜什么**：订单（按 单号 / 外部单号 / 付款人 / 电话）、商品（按 编码 / 名称）、期数（按期号）。
- **怎么用**：在顶栏搜索框输入关键词，下拉（AutoComplete）实时给出匹配结果，点某条即跳转。
- **点了跳哪**：订单 → 订单详情页；期数 → 该期报数页；商品 → 商品列表页并自动带上搜索词。

### 1.7 安排投递（通过订单）

收件人 / 订阅的「名册」页已下线——读者 / 单位的投递现在统一走**订单**：进入「订单管理 → 新建订单」（见 §4）录入付款方、商品与收件信息并「确认生效」，再在订单详情「关联快递明细」或「订单管理 → 按期排发」按期号同步到 ZTO-MF 发货明细；邮局渠道见 §2.9 邮局投递。

## 2. 周操作流程

周五执行以下步骤，为下周一的报纸做准备。

### 2.1 创建本期报数

1. 打开系统首页（`http://localhost:5173`）
2. 首页仪表盘展示以下信息：
   - **顶部统计卡片**：总期数、待处理期数、本周印数（含较上周变动）、最新报数时间
   - **一键创建**（左侧）：显示下一期期号和出版日期，点击"一键创建本期报数"按钮创建
   - **补录其他期数**（右侧）：选择历史期号手动补录
   - **报数流程**：当前期的5步骤流程指引（创建→填写→确认→发货→导出）
   - **近期印数**：最近创建的期数列表，显示期号、出版日期、总印数、状态
   - **近期印数趋势**：ECharts 折线图展示最近5期的印数变化
   - **右侧边栏**：待处理事项、操作提示、快捷入口
3. 点击"一键创建本期报数"按钮
4. 系统自动跳转到报数编辑页面

**首页近期印数**：
- 首页"近期印数"表格展示最近创建的期数
- 点击任一期可进入详情页查看
- 删除、发货、修改等操作集中在报数详情页和发货详情页中处理

### 2.2 输入变动数据

在报数编辑页面，您会看到所有报数项目，页面结构与实际报数表格一致：

页面标题会同时显示总期号和当年出版顺序，例如 `第2648期 第十四期 报数表`，其中 `第十四期` 表示该期是当年实际出版的第 14 期，休刊周不会计入。

**版数设置**：
- 页面顶部同时显示「计划 XX 版」（来自期刊表）和「实际 XX 版」（当前期的实际版数）
- 创建期次时，实际版数自动从期刊表获取计划版数作为初始值
- 确认前可通过 InputNumber 修改实际版数（步长为 4，最小值为 4）
- 当实际版数与计划版数不一致时，会显示橙色 ⚠️ 警告标签
- 版数会影响导出的 Excel 报数表

**页面布局**：
- **顶部**：临时加印（橙色虚线框，醒目显示）
- **主表格**：按类别分组显示所有项目
- **底部**：总印数合计

**变动项标识**：
- 带 `变动` 标签的项目需要每期更新
- 带 `周` / `月` / `每年` 标签表示更新频率

**需要输入的变动项**：

| 类别 | 项目 | 数据来源 | 更新频率 |
|------|------|----------|----------|
| 北京邮发 | 外埠、本市 | 邮局周五发来 | 周 |
| 北京报零 | 东部、西部 | 周四更新 | 周 |
| 广州日报 | 零售、订阅 | 周三传真/微信发来 | 周 |
| 成都杂志铺 | — | 月更新 | 月 |
| 国图贸 | — | 每年/每两年更新 | 每年 |
| 营报传媒 | 收发室、读者、备用报 | ZTO-MF表 | 周 |
| 报社订阅自投/展示 | 上犹、高铁展示 | ZTO-MF表 | 不定期 |
| 临时加印 | — | 临时需求 | 不定期 |

**临时加印归属明细**：

临时加印支持按部门分配归属明细，具体操作如下：

1. 在临时加印卡片中输入总加印数量
2. 点击"+ 添加"按钮添加归属明细行
3. 从下拉菜单选择部门（营报传媒、财经中心、中经未来、产经中心），或选择"其他"并输入自定义名称
4. 输入该部门的加印数量和自留分发数量
5. 快递数自动计算（= 加印数量 - 自留分发）
6. 所有明细行的数量之和应等于临时加印总数
7. 确认报数后，归属明细随报数一起锁定，不可再编辑
8. 归属明细的输入采用防抖保存（1.5 秒无操作后自动保存），避免快速输入时数据被覆盖

**注意**：社用报小计不包含临时加印（临时加印在页面顶部单独显示）。社用报列表中已显示的普通条目按原始印数表顺序展示；页面未显示的“印厂留存”“报社订阅自投/展示”等条目不会额外添加。合订本条目已合并计入社用报小计。

**复合项目**：
- **营报传媒**：由 收发室、读者、备用报 三个子项自动合计
- **报社订阅自投/展示**：由 上犹、高铁展示 两个子项自动合计
- 编辑子项数值后，父项合计自动更新

**操作示例**：
1. 在"外埠"输入框中输入：`5575`
2. 在"本市"输入框中输入：`1212`
3. 依次输入其他变动项
4. 查看页面底部的"总印数"，确保数值合理

### 2.3 自动保存

- **实时自动保存**：修改任何数值后，系统会在 1.5 秒后自动保存到服务器
- **保存状态提示**：页面右上角会显示 "⏳ 保存中..." → "✅ 已自动保存"
- **无需手动保存**：数据可以分多天零散填入，随时关闭页面再回来继续编辑
- **保存失败提示**：如遇网络问题，会显示 "❌ 保存失败"，此时请检查网络后刷新页面

### 2.3.1 实时中通校验

为减少"等到确认那一刻才发现报数与发货明细对不上"的问题，系统在草稿阶段就**实时显示**两者的一致性：

- **触发条件**：报数中"中通物流公司"目的地的合计 ≠ 当期 ZTO-MF 明细 `quantity` 合计
- **显示位置**：
  - 报数编辑页：「发货目的地汇总」下方（黄色 ⚠️ Alert）
  - ZTO-MF **单期详情页**顶部：「对账 · 差值（报数−发货）」统计卡
- **单期详情页样式**：顶部「本期处理状态」4 张统计卡直接摆出 **报数 · 中通合计 / 发货明细 · 合计 / 对账 · 差值（报数−发货） / 异常状态**——一致时显示「一致」，不一致时红色「差 N 份」，并列报数 / 发货两侧数字。取代了原先仅在不一致时才冒出的黄色告警条。
- **始终显示**：单期详情页的对账卡始终在顶部（无论是否一致、是否已确认）；报数编辑页的黄色 Alert 仅在不一致时出现。
- **不进单期页也能看差值**：**期数总览**表有「对账差值」列（少发红「差 N」/ 多发红「多 N」/ 一致绿），**物流工作台**首屏按本年汇总「异常期数」张数（差值≠0 的期）。

**与"确认后追溯卡片"的区别**：
- 本节描述的是**确认前的早期预警**（黄色警告，鼓励早调整）
- 第 2.4 节描述的是**确认后的漂移追溯**（卡片对比"确认时快照 vs 当前明细"，用于审计）

### 2.4 确认报数

确认所有数据无误后：
1. 点击"确认报数"按钮
2. 系统会验证：
   - 所有变动项必须有值
   - 数值不能为负
   - 总印数不能为 0
3. 验证通过后，状态变更为"已确认"
4. 确认后，所有数值显示为纯文本（只读），无法再编辑
5. 如果当期还没有ZTO-MF，系统会自动从上一期复制，并清空"信息确认"和"发货时间"
6. 若"中通物流公司"报数合计与当期ZTO-MF份数不一致，页面会显示核查提示；请进入「物流管理」→「ZTO-MF」，选择当前期号，检查并调整明细总份数
7. 报数页顶部会显示"中通校验追溯"卡片，可直接查看：
   - 确认时快照（报数中通、发货明细、差值）
   - 当前发货明细和相对报数差值
   - 是否发生"确认后明细已变更"
8. 确认后，页面顶部会显示"发货"按钮，可直接跳到该期的「ZTO-MF」页面

**删除误建期数（管理员）**：
- 管理员可在首页列表、报数详情页或该期发货明细页点击"删除"
- 系统会弹出强确认窗口，必须手工输入当前期号后才可继续
- 删除会同时删除该期报数、发货记录、临时加印和ZTO-MF；该操作不可恢复，仅用于清理误建或测试期数

**作废（管理员）**：
- 如需修改已确认的报数，管理员可以点击"作废"按钮
- 在弹出的对话框中可填写作废原因（可选）
- 作废后，状态重置为"草稿"，可以重新编辑和确认
- 作废操作会记录在修订历史中

**修订历史**：
- 页面底部以时间轴（Timeline）形式展示该期的所有确认和作废记录
- 每条记录包含操作人、操作时间和作废原因（如有）

### 2.5 检查发货明细

发货明细（ZTO-MF）现在是**三层页面**：点左侧菜单「物流管理 → ZTO-MF」先进入**物流工作台**（本年上传 / 异常概览），再经**期数总览**下钻到某一期的**单期详情**去核对明细。（三层结构详见 §3B。）

1. 点击左侧菜单「物流管理」，点「ZTO-MF」→ 进入**物流工作台 · ZTO-MF**（本年 KPI + 最近 / 后续期数 + 最近操作记录 + 待处理提醒）
2. 点顶部「查看期数总览」（或右侧快捷操作「期数总览」）→ 进入**期数总览**（全部年份，每期一行：状态 / 报数份数 / 发货份数 / 对账差值 / 最后更新时间）；可用顶部 5 张卡（全部 / 已上传 / 待上传 / 异常 / 草稿）快速筛选定位
3. 在目标期那一行点「进入详情」（或直接点工作台「最近期数」里的某期）→ 打开该期**单期详情**
4. 单期详情页会显示：
   - 顶部「本期处理状态」统计卡：报数 · 中通合计 / 发货明细 · 合计 / **对账 · 差值（报数−发货）**（一致 / 差 N 份）/ 异常状态；下方另有「是否已上传 / 是否有异常 / …」处理状态卡
   - 已确认期次另有"当期中通校验状态"区（两张卡）：左「**当前校验结果**」（当前一致 / 不一致 + 报数中通 / 当前发货明细 / 当前差值），右「**确认时快照与变更**」（确认时·当前发货明细·与确认时差值 + 是否「确认后明细已变更」）；卡内「**重新校验**」刷新校验、「**查看变更记录**」弹窗列出本期人工修改 / 孤立明细
   - 发货明细表格：精简常驻列（姓名·渠道 / 签约公司 / 收件信息 / 份数 / 来源·同步 / 状态 / 操作），点每行左侧「＋」可展开查看子渠道、频率、运输方式、截止日期、发货时间、快递单号、站点、来源订单、备注等完整字段
5. 检查名单是否正确：
   - 是否包含所有应发送的收件人
   - 数量是否正确
   - 当前页合计是否与报数页中通合计一致
   - 是否有不应发送的收件人
6. 如需导出当前期的完整 ZTO-MF，点单期详情页的"导出"按钮（或在期数总览该期行点"导出"）；导出内容为所选期号的全部 ZTO-MF，不受当前筛选条件或勾选记录影响
7. 管理员如需重做某一期的 ZTO-MF，可在单期详情页点"清空本期"并二次确认；该操作只删除当前期的 ZTO-MF，不删除期号、报数数据或临时加印

**单期详情筛选**：
- 常用筛选（渠道、签约公司、状态、姓名）直接在工具栏；频率、运输方式、子渠道收在「更多筛选」气泡里
- 工具栏右侧展示当前筛选结果的记录数和总份数，便于与报数中的"中通物流公司"合计核对
- 想跨期定位"哪些期还没上传 / 有差异"，回到**期数总览**用顶部 5 张筛选卡切换

### 2.9 邮局投递（投递记录 + 每月起投明细）

邮局投递与中通（ZTO-MF）并列，是**一种投递方式**，但**按月**运作：每月给邮局一份「**当月起投**」的投递明细，给过的读者不再重复给。邮局的数据是**投递记录**（不是订单）——它们**不进「订单管理 / 客户管理」**，完整名册在「投递名册」页看全量。系统把投递记录按「起投月」归成每月一版明细，冻结存档、导出交邮局。

进入「**物流管理 → 邮局投递**」，共 6 个页签：

> **页面呈现（各页签统一）**：顶部一条**概览行**（如 投递名册「共 N 条 · 合计 M 份 · X 条未填投递单位」、收款发票「合计金额 / 到款 · 未挂单数」）；表格只留常驻列，点每行行首「**＋**」**展开**看完整字段（邮编、金额、业务员、备注等），**无横向滚动**。投诉工单顶部有「待处理 / 处理中 / 已解决」计数、改地址有「待应用 / 未匹配 / 已应用」计数，均可**点即筛**。页签标题保留小图标（📇📦📮✏️📞🧾）。

1. **投递名册**（首个页签）：邮局记录完整名册的家。点「**导入邮局明细**」上传《报纸邮局投递明细》.xlsx，系统自动识别「邮局读者明细」工作表，逐行转成**投递记录**（不造订单；产品、覆盖期取自「年度＋起/止月」、投递＝邮局）。预览显示 导入 / 重复 / 待确认 计数；按 年度+编号 去重，重复导入自动跳过（幂等）。管理员点「确认导入」入库。可按 年度 / 起投月 / 渠道 / 投递单位 / 姓名·编号 筛选搜索、导出名册。
   - 投递单位：原表写了哪个「集订分送」就标注，没写就**留空**（不自动推断）。
   - 产品认不出（非中国经营报/商学院）不拦，留原文照常导入（邮局是纯投递）。
2. **月度起投明细**：右上选「起投月」（如 2026-07），点「**生成当月明细**」。系统收集该月起投的投递记录，冻结成一版明细。左侧按「年-月」列出各版（草稿 / 已生成 / 已发·冻结 ＋ 行数），点开看右侧清单（收报人 / 地址 / 份数 / 起止月 / 投递单位 / 渠道）。草稿 / 已生成可重建（幂等）。
3. **导出交邮局**：点「**导出 Excel**」下载 `邮局投递明细_YYYY-MM.xlsx`，发给邮局。
4. **标记已发**：交付后点「**标记已发**」→ 该版**冻结**，之后即便修改投递记录也不影响这一版（改动留痕在记录侧）。

> 说明：邮局投递记录**不进「订单管理 / 客户管理」**（它们是投递数据、不是订单），也**不进中通发货明细**——邮局的按期投递由各地集订分送自行完成。月中错过当月截止的新订单，顺延进下一版明细（起止月的「实际起月」会体现）。有平台订单号且平台对得上时（将来补），投递记录可挂到真实订单。

### 2.9.1 投诉工单

邮局投递页的「**投诉工单**」tab：点「**导入投诉**」上传含《邮局年投诉》工作表的 Excel（可与读者明细同一个工作簿）。每条投诉按 **年度＋编号** 自动关联到对应读者（投递记录）（列表「读者」列显示「**已关联读者 / 未匹配**」；关联不上仍照常记录）。处理情况（转各地邮局 11185 / XX 局）自动归一。投诉工单采用**三态处理流程**：**待处理（open）→ 处理中（in_progress）→ 已解决（resolved）**，状态由最新一次处理自动驱动。点行尾「**处理**」打开处理抽屉，填写「处理过程 / 回访结果」并提交，每次提交都会记入一条独立处理记录（时间 / 处理人 / 处理过程 / 回访结果），处理次数自动 +1；抽屉里以**时间线**展示该工单历次处理记录。可按 年度 / 状态 / 处理次数 / 收报人·编号 筛选搜索。

### 2.9.2 改地址工单 + 回访

- **改地址**：「**导入改地址**」上传《邮局年改地址》。每条按 编号＋修改日期年份 关联读者。管理员可在行尾点「**应用新地址**」，把新姓名/电话/地址写回该读者的投递记录 —— 之后生成的月度明细即用新地址（已应用的显示「✓ 已应用」，不可重复；若该记录也挂了真实订单会一并更新订单收报人；未关联到读者则先导入名册再应用）。可按 年度 / 应用状态 / 姓名·编号 筛选。
- **回访**：「**导入回访**」把读者明细里「按天开列」的回访列（如 `20240227回访`）**拍平**成一行一条（日期 / 批次 / 结果），取代原来一天加一列的做法。可按 年度 / 姓名·编号 筛选。

### 2.9.3 收款发票

「**收款发票**」tab：「**导入收款发票**」上传《提现发票合集》。每条记 金额 / 手续费 / 到款金额·日期 / 开票金额 / 抬头 / 税号 / 普专票 / 平台（抬头·税号从「发票信息」自动解析）。挂订单**优先用原始订单号**（精确）、没有时**按姓名兜底**（唯一命中才挂，否则「未挂」——等补上订单号重导即精确）。可按 平台 / 票种 / 是否挂单 / 姓名·抬头·订单号 筛选。（并入现有「财务→发票工作台」是后续，等订单号补齐、链接可靠后再做。）

> **补充：以上各源台账均可页面内手工增删改** —— 投递名册、投诉工单、改地址、回访、收款发票这 5 张台账，除了「导入 Excel」外，现在都支持在页面里直接「**新增 / 编辑 / 删除**」（写操作仅管理员可用），不必再回到 Excel 改后重导。其中**投递名册的删除有守卫**：若某条投递记录已被尚未发出的月度明细批次引用，删除会被拦下（返回 409），需先处理该批次。

## 2B. 往期数据导入

如果系统上线前已有历史期次数据，可通过「往期导入」功能批量补录，无需逐项手动输入。

> **限制**：报数文件需使用系统提供的模板或可识别的原始报数表；中通发货文件支持系统模板，也支持原始中通多工作表格式。原始报数表如果存在“临时加印”或“财经中心加印”等部门加印项，预览页会显示可编辑的「临时加印归属明细」，补齐并对上数量后可直接确认导入；总印数不一致或未命中映射项仍会在预览阶段阻断。导入成功后，仍需在报数编辑页完成复核、确认和导出，不会绕过现有流程。

### 2B.1 下载导入模板

1. 进入「**历史印数期数**」页面，点击右上角的「**导入往期**」按钮，进入往期导入页
2. 点击「**下载印数导入模板**」获取 `印数导入模板.xlsx`（含 `基本信息` / `报数项` / `临时加印明细` 三个 sheet）
3. 点击「**下载中通发货导入模板**」获取 `中通发货导入模板.xlsx`（含 `基本信息` / `发货明细` 两个 sheet）

### 2B.2 填写模板

**印数导入模板**：
- `基本信息` sheet：填写期号、出版日期（YYYY-MM-DD）、版数、备注；sheet 内附带了填写说明、临时加印规则和示例，便于其他同事直接照着填
- `报数项` sheet：按模板中的分类编码和项目名称，在「数值」列填写各项报数
- `临时加印明细` sheet：如有临时加印归属，填写部门、加印数量、自留分发数量

**中通发货导入模板**：
- `基本信息` sheet：填写与报数模板相同的期号
- `发货明细` sheet：按列填写每条发货记录（渠道、收件人姓名、地址、电话、数量等）

**原始印数表**：
- 可直接上传历史原始报数表，无需整理成系统印数导入模板
- 当前识别的工作表包括：`北京印厂`、`零售渠道\``、`订阅渠道\``、`社用报\``、`收发室自留分发（需打印）`
- 系统会自动识别“杂志铺 → 成都杂志铺”等固定映射，并在预览前校验总印数
- 如果存在“临时加印”或“XX加印”部门加印项，系统会在预览页要求手动分配归属明细，并尽量按原表部门预填；如果原表总数与映射后总数不一致或出现未映射项目，系统会明确提示差额、项目名称和数量

**原始中通多工作表文件**：
- 可直接上传历史ZTO-MF原始文件，无需整理成系统单表模板
- 当前识别的工作表包括：`每周（对公）`、`每周（读者）`、`高铁展示`/`北京悦途出行（高铁）`、`上犹`、`停发-双周（读者）`、`月底-整月`
- 系统会从标题中的“总第XXXX期”识别期号，并按各工作表表头转换渠道、子渠道、签约公司、频率、运输方式、备注等字段

### 2B.3 上传并预览

1. 在往期导入页的「**第二步：上传填写好的文件**」区域
2. 将印数文件拖入左侧上传框（或点击选择）
3. 将发货文件拖入右侧上传框
4. 点击「**预览导入**」，系统自动执行以下校验：
   - 两份文件期号是否一致
   - 目标期号是否已存在于系统
   - 报数项是否与系统模板定义匹配
   - 原始印数表是否存在需手动分配的临时加印、总印数不一致或未命中映射项
   - 中通发货文件是否属于系统模板或可识别的原始多工作表格式
   - 报数中的“中通物流公司”合计是否与ZTO-MF份数一致
5. 校验结果以卡片形式展示：
   - 期号、出版日期、**版数**、报数条目数、临时加印数、发货明细数
   - 需要手动处理的临时加印归属明细（如有，可直接在预览页新增、删除、修改部门和数量）
   - 错误列表（如有）或「数据验证通过」提示
   - **版数提示（warning）**：当印数表的版数与刊期表登记的版数不一致时，预览卡片会显示一条黄色提示，告知导入后将以印数表为准自动更新刊期表的版数

### 2B.4 确认导入

校验通过后：
1. 点击「**确认导入**」按钮
2. 如预览页提示临时加印需要手动分配，先确认明细合计等于提示的待分配份数，再点击「确认导入」
3. 系统一次性生成该期的草稿数据（报数、临时加印、ZTO-MF）
4. 如印数表的版数与刊期表不一致，系统会在导入的同一事务里把刊期表对应行的版数同步为印数表里的实际版数；导入成功的 toast 提示会显示"刊期表版数已从 X 版更新为 Y 版"
5. 自动跳转至该期的报数编辑页

**后续操作**：与正常创建的期数完全一致：
- 复核报数数据，按需修改
- 确认报数（Step 3）
- 检查并调整ZTO-MF
- 导出 Excel（Step 5）

**注意事项**：
- 如校验未通过，请按错误信息修正 Excel 文件后重新上传
- 如果提示“中通物流份数不一致”，请先核对报数文件和中通发货文件；系统不会提交不一致的数据，避免导入后再污染数据库
- 点击「重新上传」可清除当前预览结果，重新操作
- 导入会话有效期约 15 分钟，超时后需重新上传预览
- 导入页不支持直接编辑数据，所有修改请在导入后的报数编辑页进行

## 2C. 商品管理（电商导入的基础）

商品库是一张「把电商商品名翻译成订单信息」的对照表。导入电商订单前，先确保你卖的每个商品在这里有一行。**新出一个促销 = 加一行，不用改程序。**

### 2C.1 进入商品库

左侧菜单一级入口「**商品管理**」（`/products`）。表格每行是一个电商在卖的商品。首次使用如为空，可让管理员调用一次种子接口 `/api/admin/seed`，会预置 3 个已知的 CBJ 商品。

### 2C.2 新增 / 编辑商品

点「**新增商品**」（或某行「编辑」），填写：

- **商品名称**：填电商里的原始商品名（如「《中国经营报》全年订阅-618促销活动」）
- **商品编码**：唯一标识（如 `CBJ-SUB-1Y-PROMO`），建后不常改
- **别名**：把同一商品的活动后缀变体归一（如「618促销活动」「双十一订阅优惠」），回车添加；导入时这些变体都能识别到同一商品
- **是否套餐**：两刊合售（如「中国经营报和商学院」）时打开——顶层「投递方式」会**隐藏**，改在**套餐组件**里**逐刊设投递（每刊必选，无共用默认）**：每个刊物一行，选投递 + 给「固定价」或勾「拿余额」。例：中国经营报 邮局投递 + 固定 ¥240；商学院 ZTO-MF 中通 + 勾「拿余额」；¥576 自动拆成 240 + 336，两刊各按各的投递走
- **刊物 / 履约类型 / 订阅期限 / 投递方式（默认）**：下拉选
- **覆盖期算法**：订阅选「按起投月算」，单期选「最新一期」
- **参考价**：仅用于对账提示——实际入账以客户**真实付款**为准
- **状态**：启用 / 停用（停用的商品不参与导入识别）

保存后，这个商品的所有订单导入时都会自动识别。

> **停用 vs 删除**（列表每行「操作」列：编辑 / 停用·启用 / 删除）：**停用** = 保留记录、不再参与导入识别（随时点「启用」可恢复）；**删除** = 彻底移除该行、不可恢复（点删除会二次确认），适合清掉测试 / 建错的商品。两者都**不影响已导入的订单**——订单明细是独立快照，删商品不动任何历史订单。

## 2D. 电商订单导入（CBJ 小程序 / 淘宝，自动识别平台）

把 CBJ 小程序 / 淘宝 导出的订单 Excel 一次性导入系统（上传按表头**自动识别平台**）。左侧菜单「**订单管理 → 电商导入**」（`/orders/import`）。

> 📋 **页面顶部有「电商导入 · 业务规则说明」折叠面板**：列明本流程的全部业务规则（定价取实付、套餐拆分、覆盖期、转中通、状态导入策略、去重、历史归档、活动标签 / 赠品、活动订单统计口径等）。同一面板也在「**订单列表**」页顶部——看不准某条规则时点开即可。

### 2D.1 选择导入模式与起投设置

- **近期订单（要安排投递）**：会显示「本批起投设置」——填**邮局起投月**、**中通起投月**、**截止日**（此日后付款的单自动顺延到下一个月起投）。这些时间是你**人为按批设定**的，不是写死的。
- **历史归档（只补记录）**：用于补录一两年的历史订单——保留下单日期、只记客户/金额/商品，订期可留空（之后在订单页补填），**默认不自动进发货同步**（补齐订期/期号后仍可手动触发同步），无起投设置面板。

**活动标签（两种模式都可填）**：给这一批填一个标签，如 `2026-618`。它会写到这批每张订单上，方便日后**追溯**和**按活动统计**（订单列表能按活动筛选、查看）。不是活动批次可留空。

**活动赠品（仅近期模式）**：每年活动的赠品可能不同（CBJ 导出里没有这条信息，靠你按约定补）。在「活动赠品」里设：

- **订期延长**：如送 1 个月 → 这批订阅的覆盖期自动顺延 1 个月（变 13 个月）
- **赠送刊物 + 说明**：如选「商学院」+ 填「《商学院》2-3月合刊（2026-618）」→ 每张**含订阅**的订单自动加一条 **🎁 免费赠品明细**（收件人同主单，完整记录、可追溯）

> 赠品和延长**只给含订阅的订单**，单期（最新一期）不送。商品库**不用按年加新行**——价格走实付、618 基础履约稳定，活动差异全落在订单批次上。

### 2D.2 上传并预览

1. 把 CBJ 导出的 `.xlsx` 拖进上传框
2. 点「**预览导入**」，系统逐单识别，每单给出结果：
   - **✅ 导入**：识别成功，显示刊物/类型/投递/金额/覆盖期
   - **⏭ 跳过**：状态非成交（待付款 / 已取消）
   - **♻ 重复**：订单号已在系统里
   - **⚠ 待确认**：商品库找不到这个商品，或缺时间
3. 顶部显示各类计数；**投递→中通** 的单会高亮提示你核对（运费补拍转中通的情况）；金额异常会有黄色警告；赠品明细带 **🎁 赠品** 标记

> 💡 **商学院月刊单期自动识别**：标题形如「2026年X月刊《…》」「2026年2~3月合刊《…》」的单子，系统在导入时会自动识别为**商学院单期**（无需为每一期手动「加入商品库」），直接归到 **✅ 导入**。只要带「月刊 / 合刊」字样且标题里没有「中国经营报」就会被认出来；带日期的中国经营报单子仍会照常排队等你处理。真正没见过的商品（如「2026年1月新春礼包」）依然进 **⚠ 待确认**，需要你手动补全。

> 💡 **商学院单期补期次**：淘宝多商品单里的商学院单期，导出**没有分册名、看不出是哪个月**，系统标黄「**商学院单期：请补该单期次（导出无分册名）**」，并在预览里**每个商学院单期 SKU 那一行各自提供一个「补期次」输入框**——按 `YYYY-MM`（单月，如 `2026-06`）或 `YYYY-MM~MM`（合刊，如 `2026-02~03`）填即可。**一单里有好几个单期（各是不同月），就逐行分别填**，各归各、不会串。和往期期号一样，**选填、不卡流程**：不知道就先不填、照样能确认导入，导入后仍可在订单详情补。（期次用于「活动订单统计 → 按期发行量」的商学院按期口径。）

> 💡 **「最新一期」自动判到哪一期**：中国经营报每周一出刊，在售的"最新一期"每周五晚约 10 点翻到下周一那期。导入时系统按**每张单的付款时间**自动判它买的是哪一期（填好期号，预览里显示「第 XXXX 期」）。只有**周五晚翻期点前后约 4 小时**下单的"临界单"会**标黄提示你核对**（因为 10 点只是大概时间），其余全自动。

> 💡 **往期零售的期号**：往期单卖的是哪一期，是客户**联系客服**单独告知的、**不在订单里**，系统没法自动判。导入时这种单会**标黄「请补该单实际期号（客服确认）」**，并在预览里**每个往期单期 SKU 那一行各自提供一个「补期号」小输入框**——你按客服记录当场填上即可；**一单买了几份不同期，就逐行分别填**，各归各、不会串。**补期号是选填**：不知道就先不填，照样能「确认导入」，导入后再到该订单「**编辑 → 单期期号**」补也行（单期最终必填）。加入商品库时智能默认已能把"往期 / 零售"识别成 **单期 + 自定义**。
4. **待确认商品就地补全**：预览页顶部「**待确认商品汇总**」把未识别订单按商品名聚合（如 98 单≈十几种），每种一个「**➕ 加入商品库**」——点开是**预填好名称、带智能默认**的快速新增表单，你改下参考价/编码即可保存；保存后**自动重新预览**，用到它的订单一起变「✅ 导入」。逐个加完即可全部识别，不用来回切页面。
5. **点任意行看详情**：点预览表任一行弹出抽屉看该单明细；待确认行点开就是上面的快速新增。

### 2D.3 确认导入

- 确认无误后点「**确认导入 N 单**」，系统一次性把可导入的单建成订单（生效状态、自动分配订单编码、商业状态/活动标签/赠品按设置写入）
- 跳过/重复/待确认的单不会建单
- 个别订单的起止日期、状态、赠品等可在导入后到订单详情页再调整
- 导入后到「**订单管理 → 订单列表**」，用顶部「**活动**」筛选框输入标签（如 `2026-618`）即可查这次活动的所有订单（按活动统计的基础）

> **小贴士**：导入会话有效期约 30 分钟，超时需重新预览。同一份文件重复导入会自动按订单号跳过已存在的单，不会重复建单。

## 3. 物流管理

### 3.0 报数模板

报数模板定义每期《报数》包含哪些项目——新建报数时按这张母表逐行生成。模板的变更仅影响新创建的期数，已有期数不受影响。

1. 在左侧菜单展开「印数管理」，点击「报数模板」
2. 页面顶部有 4 张统计卡：模板项目数、类别数、变动项数、默认值合计（即新建报数的基线印数）
3. 模板**按类别分组**展示（北京邮发 / 北京报零 / 广州日报 / 成都杂志铺 / 国图贸 / 社用报 / 合订本），每个类别组头显示该渠道的**报数频率**（每周 / 每月 / 每年）、项数与默认值小计，与报数编辑页的报表结构一致
4. 顶部**类别快筛**可只看某一类；每项标注「变动 / 固定」（变动 = 每期需重新填写，固定 = 常年不变）
5. **新增项目**：点右上角「新增项目」，或在某类别组头点「在此类别新增」（自动预填类别）；填写类别、子类别、显示名称、默认值、是否变动项、排序
6. **编辑 / 删除**：点操作列的编辑 / 删除按钮
7. **调整顺序**：直接拖动每行左侧的手柄 ⠿，在同一类别内拖放即可改排序（自动保存）

**注意**：删除或修改模板不会影响已创建的期数数据。

### 3.1 收件人名册（已下线）

> ℹ️ 原「物流管理 · 收件人」页面（收件人 + 订阅名册的手工增改删）**已下线**。经核查，收件人 / 订阅数据不参与当前的发货流程——**发货明细（ZTO-MF）统一由订单驱动生成**（见 §4 订单管理 与下方 §3B）；历史的收件人 / 订阅数据仅在后台保留，不再有页面入口。

「物流管理 → ZTO-MF」自 2026-07 起重构为**三层页面**，让你不用逐期点击就能看到"哪些期已上传 / 未上传 / 异常"，再下钻处理具体发货数据：

### 3B.1 物流工作台（进入 ZTO-MF 的首屏）
点菜单「ZTO-MF」即到，聚焦**本年**的上传与对账健康度：
1. **本年 4 张 KPI**：已上传期数 / 待上传期数（含"未创建"）/ 异常期数（对账差值≠0）/ 本月最新更新，每张标注"占本年 X%"
2. **期数状态总览**：列出「最近期数」和「后续期数」（各取最近 5 期、单行展示），点某期直接进单期详情；右上「查看全部期数」进期数总览
3. **最近操作记录**：时间 / 操作人 / 操作内容 / 期数 / 状态（成功·失败），汇总近期发货明细相关写操作
4. **右侧「待处理提醒」**：尚未上传发货明细 / 报数与发货差异 / 草稿未确认 三类待办计数
5. **右侧「快捷操作」**：期数总览 / 数据报表 / 印数报数 一键直达

### 3B.2 期数总览（逐期纵览）
从工作台「查看期数总览」进入（路由 `/logistics/issues`），按期把 报数 / 发货 / 对账 / 状态一次看全：

### 3B.3 单期详情 · 查看发货明细

1. 顶部**本期处理状态**：报数 · 中通合计 / 发货明细 · 合计 / 对账 · 差值（报数−发货）/ 异常状态 四张卡，下方另有「是否已上传 / 是否有异常」等处理状态卡

3. 使用下拉筛选器按条件查看：
   - **渠道**：渠道订阅、对公订阅、个人订阅、记者站、赠阅（子渠道：监管/政府）、库房留存、报社留存
   - **运输方式**：中通物流、邮政物流、包车运输、库房留存（以彩色标签显示）
   - **频率**：周、半月、月
   - **状态**：正常、停发
4. 使用搜索框按收件人姓名模糊搜索；「操作」列固定在表格右侧，表格内容过宽时可横向滚动

### 3B.4 新增发货记录

2. 点击该行的删除图标（鼠标悬停显示"删除"提示）
3. 在确认弹窗中点击「确定」

## 4. 订单管理（V1.3）

订单管理用于记录读者订单，从下单到生效再到作废的全流程，并跟踪订单覆盖期内还剩多少期未发。当前版本支持手工创建订单、订阅套餐价、确认生效后继续就地编辑订单明细/履约目标，并可在单张订单详情中按期号手动预览 / 应用同步到 ZTO-MF 发货明细（`shipping_details`），并支持电商订单批量导入（CBJ 小程序 / 淘宝，上传自动识别平台，见 §2D）。财务对账和客户自助下单仍排在后续规划中。

入口：左侧菜单「订单管理」→ 「订单列表」 / 「新建订单」。

### 4.1 如何创建一张订单

1. 点击侧边栏「订单管理 → 新建订单」（或在订单列表页右上角点击「新建订单」）
2. 在「基本信息」卡片填写订单基础字段：
   - **付款主体名称** ⭐：付款方姓名或机构名称（自由文本，系统不区分类型）
   - **付款主体联系方式**：电话或邮箱（可选）
   - **订单日期** ⭐：实际下单日期
   - **付款方式**：微信 / 支付宝 / 银行卡 / 对公转账 / 现金 / 冲抵 / 其他（可选）
   - **收款人**：经办人姓名（可选）
   - **总金额 / 已付金额**：可选，DECIMAL(10, 2)
     - V1.1 范围限定**个人客户预付 + 同事赠阅**两种场景。个人预付时 `总金额 = 已付金额`；赠阅时（履约类型选「赠阅」+ 计费类型选「免费赠阅」）两者都填 0
     - "渠道订单（含先履约后付款 / 赊账）"当前尚未进入财务闭环，留待后续版本加「收款流水 + 欠款追踪 + 未付清筛选」时再覆盖
   - **是否需要发票 / 发票抬头 / 纳税人识别号 / 发票接收邮箱**：勾选"是否需要发票"后建议把后三项一起填齐
     - **发票抬头**：购方名称（个人发票填本人姓名；企业发票填企业全称）
     - **纳税人识别号**：统一社会信用代码（USCC，常见 18 位字母数字）；个人发票可留空
     - **发票接收邮箱**：电子发票推送地址；填写后会单独保存，不与备注混用
   - 「外部订单号」：用于后续与电商对接，可不填
   - **来源平台 / 来源店铺**：下拉选择，1:1 联动（仅当订单从这 3 个渠道来）
     - 微信小程序 → CBJ+
     - 淘宝 → 中国经营报发行部
     - 有赞 → 中国经营报微店
     - 选好平台会自动填上对应店铺；切换平台时店铺会跟着更新
     - 老数据中的非标值（如"天猫"）会原样展示，不会丢，但无法在下拉里再次选中；如需保留请保留，需要规范则手动改为以上 3 项之一

   > 💡 **关于"录入方式"**：通过本表单创建的订单都是手工录入，因此该字段不出现在表单里——系统会自动标记为「手工录入」，在订单详情页右上角以 `📥 手工录入` Tag 形式展示。电商订单导入建的单会自动标记 `excel_import`（见 §2D）；未来「电商 API 同步」入口会启用 `api_sync`。录入方式始终由后端按入口自动设置。
3. 在「订单明细」卡片至少添加一条明细，每条明细需填写：
   - **履约类型** ⭐：**订阅 / 单期 / 赠阅 / 补寄 / 续订 / 换订**（后端枚举值 `subscription / single_issue / gift / makeup / extension / replacement`，不同类型字段要求不同）
   - **计费类型**：**付费 / 免费赠阅 / 搭赠**（赠阅明细通常选「免费赠阅」；搭赠用于"买 X 送 Y"场景）
   - **订阅期限**（仅订阅 / 续订显示）：**半年 / 一年 / 自定义**
     - 选「半年 / 一年」后，需要继续选择**起始月份**和**投递/收费方式**，系统会按刊期表自动换算实际覆盖开始/结束日期
     - 选「自定义」后，可手工填写覆盖开始 / 结束和单份套餐价，适用于非标周期
     - 半年 / 一年场景按实际出版期覆盖范围计算，休刊不补偿、不顺延
   - **投递/收费方式**（仅半年 / 一年显示）：**邮局投递 / ZTO-MF 快递**
   - **起始月份**（仅半年 / 一年显示）：格式为 `YYYY-MM`，用于从刊期表换算起订周期
   - **总份数** ⭐：**每期**寄出的份数（与覆盖期长度无关）
     - 订阅：等于「订户数 × 每订户每期份数」，常见每订户每期 1 份 → 2 个订户即总份数 2
     - 单期/零售：本期总共寄出的份数
   - **单价** ⭐：每「份」对应的价格；订阅类在半年 / 一年模式下会根据套餐自动带出：
     - 邮局投递：半年 120 元，一年 240 元
     - ZTO-MF 快递：半年 195 元，一年 390 元
     - 自定义 → 手工填写「订阅单价 / 户（按覆盖期）」
     - 单期/零售 → 「单价」（每份零售价，如 5 元/份）
   - **小计**：自动 = 总份数 × 单价（公式与期数无关）
   - **覆盖开始 / 结束**：订阅 / 续订必填；半年 / 一年场景由系统自动预览并回填，自定义可手工编辑；单期不需要
   - **期号**：仅单期订单必填且唯一
   - **创建期数（自动）**：保存时由系统根据覆盖期 + 刊期表 + 是否休刊算出

**订阅期限与套餐价**

新建订阅类订单时，先选择「半年 / 一年 / 自定义」。
- 半年 / 一年：选择起始月份和投递/收费方式后，系统自动从期刊表计算实际覆盖期、预计发货期数、单份套餐价和应收小计。
  - 邮局投递：半年 120 元，一年 240 元。
  - ZTO-MF 快递：半年 195 元，一年 390 元。
- 自定义：手工填写覆盖期和单份套餐价。

「每期总份数」是每一期要寄出的份数，不是整个订阅周期的报纸总量。

4. 每条明细下方有一个「履约目标」表格，至少录入一行收件人，每行需要：
   - **收件人姓名** ⭐ / **电话** ⭐ / **地址** ⭐ / **邮编**
   - **份数** ⭐：该收件人**每期**收到的份数（订阅情况下一般为 1）。所有目标的份数合计必须等于该明细的「总份数」，否则保存时会被卡住
5. 点击底部「保存草稿」：通过校验后保存，自动跳转到该订单的编辑页（URL 带 `id`），允许继续修改
6. 点击底部「确认生效」：自动先保存再调用 confirm，订单进入 `active` 状态，系统分配订单号（`ORD-YYYY-NNNNNN`，年内 6 位序号），并写入期数快照

> 💡 草稿状态下可以反复修改所有字段；确认生效后，普通基础字段仍按白名单直接修改，**明细和履约目标也可以通过“编辑”页继续调整**。系统会要求填写“新版本生效起始期号”，并在后台自动保留旧履约方案版本，避免覆盖历史。

### 4.2 如何编辑已生效订单的明细

1. 在订单列表或详情页点击右上角「编辑」进入订单编辑页
2. 如果订单状态为「生效」，在「订单明细」卡片顶部先填写：
   - **新版本生效起始期号**：表示这次收件信息/份数变更从哪一期开始生效
   - **变更原因**：可选，建议填写为“客户换地址”“补加一位收件人”等，便于事件流审计
3. 在明细列表中直接执行以下操作：
   - **修改已有明细**：保留原明细，调整覆盖期、份数、单价或收件人信息
   - **新增明细**：点击「添加明细」，录入新的履约类型和目标列表
   - **删除明细**：移除某条已生效明细；系统不会物理删除历史，而是把它标记为取消并关闭当前 allocation 生效区间
4. 点击底部保存按钮后：
   - 没有 `id` 的明细会作为新明细写入
   - 目标列表有变化的旧明细会自动生成新的履约方案版本
   - 未出现在本次提交中的旧明细会标记为取消
5. 返回订单详情页的「履约方案」和「事件流」Tab，可查看新旧版本、变更原因以及 `item_added / item_removed / item_modified` 事件

### 4.3 如何查看订单进度

1. 点击侧边栏「订单管理 → 订单列表」
2. 顶部筛选器支持按：
   - **状态**（草稿 / 生效 / 作废）
   - **付款主体名称**（模糊）
   - **覆盖期范围**
   - **是否存在偏差**（创建时期数与当前预估不一致）
   - 列表中「渠道/平台」列展示 `source_platform`（如"微信小程序"/"淘宝"/"有赞"），可悬停查看店铺名
3. 列表中点击任意行进入订单详情页，包含 4 个 Tab：
   - **明细**：每条明细的履约类型、覆盖期、当前进度（创建时期数 / 当前预估 / 已同步 / 偏差），偏差以颜色提示（红 = 减少、橙 = 增加、绿 = 一致）
   - **履约方案**：每条明细的分配方案版本与目标列表
   - **关联快递明细**：按期号预览并同步订单目标到 ZTO-MF，查看创建 / 更新 / 跳过 / 冲突结果
   - **事件流**：完整的操作时间线，包括创建、确认、修改、作废、目标变更等
4. 详情页右上角可以直接跳转回「编辑」（草稿 / 生效状态可见）或「作废」

### 4.4 如何将订单同步到 ZTO-MF

1. 前置条件：订单必须已经「确认生效」并处于 `active` 状态；草稿或作废订单不能同步到 ZTO-MF。
2. 进入订单详情页，打开「关联快递明细」Tab。
3. 选择要同步的期号，点击预览同步。
4. 检查预览结果中的创建、更新、跳过和冲突明细；确认没有冲突后，点击确认同步。
5. 同步成功后，进入「物流管理」→「ZTO-MF」，选择相同期号，可看到系统生成的发货明细，来源显示为「订单生成」。
6. 如果预览出现冲突，先在 ZTO-MF 中处理手工修改、重复关联或异常明细，再回到订单详情重新预览。

### 4.5 如何作废订单

1. 进入订单列表，或在订单详情页右上角，点击「作废」
2. 在弹出的对话框中**填写作废原因**（必填，会记入事件流）
3. 点击「确认作废」：订单状态变为 `void`，进入终态，不可恢复
4. 作废后订单仍可查看，但所有编辑入口均被关闭

> ⚠️ 作废是终态。如果只是想改字段或收件分配，优先通过编辑页完成；只有当整张订单业务上失效（如整单退款、客户取消）时才需要作废。

### 4.5b 如何删除订单（管理员）

删除用于**清理草稿 / 误建 / 测试遗留**订单——把数据从库里彻底移除（区别于「作废」的软标记留痕）。仅管理员可见、可操作。

**可删条件（同时满足）**：
- 状态是**草稿（draft）**或**已作废（void）**——生效（active）订单**不显示**删除按钮，需先「作废」再删；
- 该订单**未生成任何 ZTO-MF 发货明细**——一旦同步过发货明细，系统会拦下（`409`），请改用「作废」停发。

**单行删除**：在订单列表「操作」列点红色「删除」→ 二次确认弹窗 →「确认删除」。
**批量删除**：勾选多单后，顶部批量区点「批量删除 (N)」——N 只统计选中项里**可删**的单；不可删的（生效 / 已发货）会自动跳过，弹窗里会提示跳过数量。

删除会**级联移除**该订单的明细 / 收款 / 事件 / 发票 / 退款记录（订单私有数据），而邮局投递等真实业务记录会保留、仅解除与订单的关联。每次删除都会写入**操作日志**（工作台「最近操作记录」可见）。

> ⚠️ 删除不可恢复。日常业务失效请用「作废」（留审计痕迹）；「删除」只用于清理不该留在库里的草稿 / 测试数据。

### 4.6 订单功能版本状态

**已完成**：

- ✅ 电商订单批量导入（CBJ 小程序 / 淘宝，上传自动识别平台）+ 商品库 + 活动标签/赠品：上传 Excel → 预览 → 确认批量建单，详见 §2C、§2D。

### 4.7 活动订单统计

入口：左侧菜单「**订单管理 → 活动订单统计**」（`/analytics`）。用于按活动和按期回看销量与销售额，做活动复盘和单期销量核对。

> 统计**只计有效订单**（已确认 / 导入生效的单）；草稿、待确认、作废的单**不计入**，所以这里的数字是真实成交口径。

页面顶部有一个 **下单日期** 范围筛选器，两张表都受它控制——选好起止日期，两张表同时按这个区间重新统计。

**按活动统计**：每行是一个活动标签，列含：

| 列 | 含义 |
|------|------|
| 活动 | 订单上的活动标签（如 `2026-618`） |
| 订单数 | 该活动下的有效订单数 |
| 原价合计 | 各单原价（CBJ 导出的「原价」）的合计；没抓到原价的单按实收计 |
| 实收金额 | 各单实际付款的合计 |
| 折扣 | 让利多少，显示「省 ¥X」及对应百分比（百分比 =（原价合计 − 实收）÷ 原价合计） |

> 只有**带活动标签**的订单才会出现在这张表里；没打标签的单不计入按活动统计。「折扣」帮你一眼看出这次活动整体让了多少利——省 ¥ 越大、百分比越高，让利越多。

**按期统计**：每行是某刊物的某一期，列含：

| 列 | 含义 |
|------|------|
| 刊物 | 刊物名称 |
| 期次 | 期标识（`issue_label`，如 `2026-03`、`2026-02~03`），主要是商学院月刊 |
| 销量（份） | 该期卖出的总份数 |
| 销售额 | 该期销售金额合计 |
| 行数 | 对应的明细行数 |

> 按期统计看的是**带期次标识的单期明细**（主要为商学院月刊单期），方便核对某一期到底卖了多少份。可在筛选里指定刊物只看某一刊。

**按期发行量（商学院 · 单期 + 订阅）**：回答「某期实际发 / 印多少本（**含订阅订户**）」。每行是商学院某期，列含：

| 列 | 含义 |
|------|------|
| 期次 | 商学院刊历里的期（`2026-01`、`2026-02~03` 合刊）|
| 单期销量 | 单买这期的份数 |
| 订阅覆盖 | 覆盖该期的订阅份数（订阅按覆盖期落到刊历展开、合刊只计一次）|
| 合计发行 | 单期 + 订阅 |

> 与「按期统计」的区别：按期统计只数**单期零售**；这里**含订阅订户**（全年 / 半年 / 季度订阅覆盖到该期的都算）。右上角可切年份。缺覆盖期的订阅会单独提示「未展开 N 张」（历史 / 未配起投月的单，补齐覆盖期后即计入）。商学院刊历存在 `bs_issues` 表，每年加几行维护（哪些月单刊、哪些合刊）。

## 5. 常见问题 FAQ

### Q1: 如何修改已确认的报数？

**A**:
1. 管理员可以在报数编辑页面点击"作废"按钮
2. 在弹出的对话框中可以填写作废原因（可选）
3. 确认作废后，期数状态重置为"草稿"
4. 然后可以重新编辑数据并再次确认
5. 所有作废记录会保存在修订历史中，可在页面底部查看

### Q2: 订阅到期后还会继续发吗？

**A**:
- 现在订阅覆盖由**订单**的覆盖期（覆盖开始 / 结束）决定：当期出版日期超出订单覆盖期后，同步到 ZTO-MF 时不再为该订单生成发货明细。
- 需要继续投递，请为客户新建续订订单（见 §4）。

### Q3: 休刊期怎么处理？

**A**:
- 系统已预先配置好 2026 年的休刊日期：
  - 2026-02-16（休刊，无期号）
  - 2026-02-23（休刊，无期号）
  - 2026-10-05（休刊，无期号）
- 在"下一期信息"中，系统会自动跳过休刊日期
- 休刊周不占用期号，也不会创建期数、报数或发货明细
- 实际出版周的期号保持连续：休刊后的下一期会接续上一期已出版期号 + 1

### Q4: 半月 / 月底的发货规则是什么？

**A**:
- 这套「按收件人频率（周 / 半月 / 月底）自动决定是否发」的规则属于**已下线的收件人订阅体系**，当前不再通过页面使用。
- 现在按期投递由订单覆盖期决定，逐期在「订单管理 → 按期排发」同步到 ZTO-MF。

### Q5: 怎样批量安排收件人 / 投递？

**A**:
- 收件人名册页已下线。批量投递请走**电商订单导入**（§2D，CBJ / 淘宝 一次性建单）或**邮局明细导入**（§2.9），再按期同步 / 生成明细。

### Q6: Excel 密码忘了怎么办？

**A**:
- 所有 Excel 文件的密码统一为：`0611`
- 如果修改密码，需要在后端代码中修改：
  - 文件：`backend/app/services/excel_service.py`
  - 搜索：`sheet.protection.password = "0611"`
  - 修改为新密码

### Q7: 导出的 Excel 格式不对怎么办？

**A**:
1. 检查是否存在自定义模板：`backend/app/templates/report_template.xlsx`
2. 如果存在模板，系统会使用模板填充数据
3. 如果不存在，系统会动态创建工作簿
4. 如果格式不满足要求，可以：
   - 创建符合规范的模板文件
   - 或修改代码中的导出逻辑

### Q8: 系统支持多年管理吗？

**A**:
- 目前默认初始化 2026 年刊期表。
- 管理员可在侧边栏展开「刊期表管理」菜单，包含两个子项：
  - **期刊表**：顶部 4 张概览卡（出版期数 / 休刊次数 / 期号范围 / **异常版次**——即计划版数与实际版数不一致的期数）。其下「**年度概览**」用 12 个月圆点标出每月状态（●正常绿 / ●版次调整橙 / ●休刊灰——该月有「计划≠实际」则橙、含休刊周则灰、否则绿），点某月圆点即筛到该月。筛选工具栏支持按 月份 / **出版日期区间** / 期号 / 状态（正常·休刊）过滤，改好条件后点「**查询**」生效、「**重置**」清空。正文是「**全年排期矩阵**」：每行一个月，列为「第 1 期 … 第 N 期」（N = 当月出刊周数，一个月最多 5 周），每格显示该周的 出版日期 + 期号 + 状态·版数；**计划≠实际**的格以橙底标「实际 XX 版」，**休刊**周为虚线 ⊘「休刊」格，无该周则留「—」。最后一列「**月度状态**」用胶囊标注「N 期 / 正常（绿）· 含调整（橙）· 含休刊（灰）」。矩阵随筛选联动，列数按当前结果动态收放。
  - **导入期刊表**：管理员上传或拖拽年度刊期 PDF（须为可抽取文字的文本 PDF；扫描版需先 OCR 成可复制文字）。页面以 **4 步进度条**（上传 PDF → 解析校验 → 人工确认 → 确认保存）引导：**未上传时**（默认页）左侧「PDF 上传与解析预览」为云上传拖拽区 +「选择文件上传 / 下载模板说明」，右侧「导入说明」给出 4 步引导与提示，下方 4 张状态卡（当前年份 / 已保存期刊表 / 最近导入状态 / 最近更新时间，由上传记录派生），「上传记录」表的已保存行可「查看记录 / 再次导入」。**解析后**左侧「**PDF 上传与解析预览**」卡显示文件与「解析完成，可确认保存」状态，并提供「重新上传 PDF」「下载解析结果」（导出 CSV）；右侧「**导入摘要**」卡列出 年份 / 总行数 / 出版期数 / 休刊次数 / 期号范围 / **默认版数**（PDF 未标注时取各行版数众数兜底，可手改）/ 待人工确认项 / 当前状态，并橙色提示「保存后将覆盖该年份现有期刊表」；其下 4 张统计卡（解析记录 / 正常项 / 休刊项 / 异常·待确认）。「**人工确认预览**」区支持 **月份快筛 / 状态筛 / 搜索（日期或期号）**，按月份分组逐行核对：出版日期、期号、版数**行内可编辑**，「休刊 / 出版」开关切换，行内删除或「新增一行」；右侧「人工确认说明」侧卡给出核对步骤。**校验状态**分 正常 / 休刊 / **待确认**（行版数≠默认版数时橙色标注「版数异常，建议复核」）/ 需检查（解析硬错误）。版数从 PDF 文本中的「对开XX版」自动提取；如果 PDF 解析出其他年份，页面会自动切换到该年份上下文。
- 管理员可在预览表中手动修改出版日期、期号和休刊状态，也可新增或删除行。修改后先点击「应用修正并重新校验」，系统会更新服务端待确认预览并重新计算摘要和错误；通过校验后再点击「确认保存」。系统会弹出确认提示，说明保存后将更新该年份的刊期表（其他年份不受影响）；确认后调用无请求体的 `POST /api/schedule/uploads/{upload_id}/commit`，使用服务端预览时保存的行数据写入或替换正式刊期表，同时自动清理同年份其他待确认的上传记录，并刷新刊期表和上传记录。单个 PDF 最大 10 MB。
- 提交前请重点核对出版日期、期号连续性和休刊行；休刊行不填写期号。
- 如果上传提示缺少 PDF 解析依赖，请在后端虚拟环境中执行 `pip install -r requirements.txt` 后重启后端；如果提示无法匹配出版日期，说明 PDF 文本抽取中存在系统无法识别的日期数字，需要在预览表中手动修正，或改用可复制文字更清晰的版本。
- 如果该年份已经创建过期数，新刊期表不能删除这些期号，也不能修改它们的出版日期，系统会拒绝此类提交。
- 非管理员用户可以查看刊期表和上传记录，但不会看到上传或确认保存控件。
- 上传历史可通过 `GET /api/schedule/uploads?year=年份` 查看。
- 管理员可在上传记录列表中手动删除待确认（未保存）的记录；已保存的记录不可删除。

### Q9: 发货数量可以为 0 吗？

**A**:
- 发货明细里某条记录的数量可以手动改为 0（表示本期不发这一条）。
- 若整条记录不再需要，可直接删除，或在其来源订单里作废 / 调整覆盖期。

### Q10: 如何查看历史期数的数据？

**A**:
1. 在侧边栏展开「印数管理」，点击「历史期数」进入历史印数期数页面
2. 页面顶部有 4 张统计卡（总期数 / 已确认 / 草稿待确认 / 本年累计印数）快速概览；表格每期直接显示「印数（份）」列
3. 支持按状态**分段快筛**（全部 / 草稿 / 已确认 / 已导出）、按期号搜索、按出版日期区间筛选
4. 每期可一键直达：报数（草稿显示「去报数」）、中通明细、导出 Excel；点击整行也可进入报数
5. 也可以在首页的"最近报数"区域查看最近几期
6. 已确认或已导出的期数无法修改，但可以查看和重新导出

### Q11: 如何补录历史期次数据？

**A**: 
- 在「历史印数期数」页面点击右上角的「导入往期」按钮（见第 2B 节）
- 下载系统提供的两份导入模板，按格式填写数据
- 上传并预览校验通过后，点击「确认导入」一次性生成草稿期数
- 导入后在报数编辑页复核数据，再按正常流程确认和导出

```bash
cd frontend

```

```bash
cd backend
venv\Scripts\activate
uvicorn app.main:app --host 0.0.0.0 --port 8000
```

**注意**：
- 生产模式下，FastAPI 会自动检测 `frontend/dist` 目录
- 如果存在，会提供静态文件服务
- 访问 `http://your-server:8000` 即可使用系统

### 5.3 使用反向代理（推荐）

**Nginx 配置示例**：

```nginx
server {
    listen 80;
    server_name your-domain.com;

    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
}
```

### 5.4 进程管理（推荐）

**使用 systemd（Linux）**：

创建 `/etc/systemd/system/newspaper.service`：

```ini
[Unit]
Description=Newspaper Report System
After=network.target

[Service]
Type=simple
User=your-user
WorkingDirectory=/path/to/FirstTry/backend
Environment="PATH=/path/to/FirstTry/backend/venv/bin"
ExecStart=/path/to/FirstTry/backend/venv/bin/uvicorn app.main:app --host 0.0.0.0 --port 8000
Restart=always

[Install]
WantedBy=multi-user.target
```

启动服务：
```bash
sudo systemctl enable newspaper
sudo systemctl start newspaper
sudo systemctl status newspaper
```

## 7. 故障排查

### 7.1 后端启动失败

**症状**：运行 `uvicorn app.main:app` 报错

**可能原因**：
1. 虚拟环境未激活
2. 依赖未安装
3. 数据库连接失败
4. `.env` 文件配置错误

**解决方案**：
1. 确保虚拟环境已激活（命令行前有 `(venv)` 标记）
2. 重新安装依赖：`pip install -r requirements.txt`
3. 检查 MySQL 是否运行：`mysql -u root -p`
4. 检查 `.env` 文件中的数据库配置

### 7.2 前端无法访问后端

**症状**：前端页面显示"网络错误"或"无法连接到服务器"

**可能原因**：
1. 后端服务未启动
2. CORS 配置错误
3. 端口冲突

**解决方案**：
1. 确认后端运行在 `http://localhost:8000`
2. 检查 `backend/app/main.py` 中的 CORS 配置
3. 确认前端 API 请求地址正确（`frontend/src/api/` 目录下的文件）

### 7.3 导出 Excel 失败

**症状**：点击"导出"按钮后无响应或报错

**可能原因**：
1. `openpyxl` 库未安装
2. 数据不完整
3. 期数状态不正确

**解决方案**：
1. 确认已安装 openpyxl：`pip show openpyxl`
2. 检查该期的报数数据和发货明细是否完整
3. 确认期数状态为"已确认"或"已导出"

### 7.4 发货明细为空

**症状**：某期 ZTO-MF 发货明细为空

**可能原因**：
1. 该期还没有可同步的**生效订单**（覆盖到本期）
2. 订单未「确认生效」，或覆盖期不含当期出版日期
3. 尚未执行「按期排发 / 关联快递明细」的同步

**解决方案**：
1. 到「订单管理 → 订单列表」确认有覆盖本期的生效订单
2. 在订单详情「关联快递明细」或「按期排发」按期号预览并应用同步
3. 邮局渠道见 §2.9（不进 ZTO-MF）
