# 操作手册

## 1. 首次使用

### 1.1 环境准备

#### 安装依赖软件
- **Python 3.11+**：从 [python.org](https://www.python.org/downloads/) 下载安装
- **Node.js 18+**：从 [nodejs.org](https://nodejs.org/) 下载安装
- **MySQL 数据库**：确保 MySQL 服务正在运行

#### 验证安装
```bash
# 检查 Python 版本
python --version

# 检查 Node.js 版本
node --version

# 检查 MySQL 是否运行
mysql --version
```

### 1.2 配置数据库连接

在项目根目录创建 `.env` 文件：

```env
MYSQL_HOST=localhost
MYSQL_PORT=3306
MYSQL_USER=your_username
MYSQL_PASSWORD=your_password
MYSQL_DATABASE=newspaper_system
```

**注意**：
- 确保数据库用户有创建表的权限
- 数据库 `newspaper_system` 需要提前创建好

### 1.3 启动后端服务

```bash
# 进入后端目录
cd backend

# 创建虚拟环境（仅首次）
python -m venv venv

# 激活虚拟环境
venv\Scripts\activate  # Windows
# 或
source venv/bin/activate  # macOS/Linux

# 安装依赖
pip install -r requirements.txt

# 运行数据库迁移
alembic upgrade head

# 启动服务
uvicorn app.main:app --reload --port 8000
```

**成功标志**：
```
INFO:     Uvicorn running on http://127.0.0.1:8000 (Press CTRL+C to quit)
```

### 1.4 初始化种子数据

打开新的命令行窗口（保持后端服务运行），执行：

```bash
curl -X POST http://localhost:8000/api/admin/seed
```

**响应示例**：
```json
{
  "message": "Seeded 52 schedule entries, 30 report templates"
}
```

**种子数据包含**：
- 2026 年刊期表（52 周，49 期出版）
- 报数模板配置（固定项 + 变动项）

### 1.5 启动前端服务（开发模式）

打开新的命令行窗口，执行：

```bash
# 进入前端目录
cd frontend

# 安装依赖（仅首次）
npm install

# 启动开发服务器
npm run dev
```

**成功标志**：
```
VITE ready in 500 ms

➜  Local:   http://localhost:5173/
```

### 1.6 访问系统

在浏览器中打开 `http://localhost:5173`，系统会自动跳转到登录页面。

**登录**：
- 输入用户名和密码（默认管理员账户：用户名 `admin`，密码 `admin123`）
- 点击"登录"按钮
- 登录成功后自动跳转到系统首页

### 1.7 录入收件人信息

1. 点击顶部菜单"收件人管理"
2. 点击"新增收件人"按钮
3. 填写信息：

   - **电话**：13800138000
   - **地址**：北京市朝阳区某某街道
   - **类型**：读者
   - **频率**：周
4. 点击"保存"

重复此步骤，录入所有收件人。

### 1.8 为收件人添加订阅

1. 在收件人列表中找到刚才创建的"张三"
2. 点击"订阅"按钮
3. 填写订阅信息：

## 2. 周操作流程

周五执行以下步骤，为下周一的报纸做准备。

### 2.1 创建本期报数

1. 打开系统首页（`http://localhost:5173`）
2. 查看"下一期信息"卡片：
   - 期号：2635
   - 出版日期：2026-01-05
3. 点击"创建本期报数"按钮
4. 系统自动跳转到报数编辑页面

### 2.2 输入变动数据

在报数编辑页面，您会看到所有报数项目，页面结构与实际报数表格一致：

**版数设置**：
- 页面顶部显示当前版数，默认值为 24
- 可通过 InputNumber 修改版数（步长为 4，最小值为 4）
- 版数会影响导出的 Excel 报数表

**页面布局**：
- **顶部**：临时加印（橙色虚线框，醒目显示）
- **主表格**：按类别分组显示所有项目
- **底部**：总印数合计

**变动项标识**：
- 带 `变动` 标签的项目需要每期更新
- 带 `周` / `月` / `每年` 标签表示更新频率

**需要输入的变动项**：

| 类别 | 项目 | 数据来源 | 更新频率 |
|------|------|----------|----------|
| 北京邮发 | 外埠、本市 | 邮局周五发来 | 周 |
| 北京报零 | 东部、西部 | 周四更新 | 周 |
| 广州日报 | 零售、订阅 | 周三传真/微信发来 | 周 |
| 成都杂志铺 | — | 月更新 | 月 |
| 国图贸 | — | 每年/每两年更新 | 每年 |
| 营报传媒 | 收发室、读者、备用报 | 中通发货明细表 | 周 |
| 报社订阅自投/展示 | 上犹、高铁展示 | 中通发货明细表 | 不定期 |
| 临时加印 | — | 临时需求 | 不定期 |

**临时加印归属明细**：

临时加印支持按部门分配归属明细，具体操作如下：

1. 在临时加印卡片中输入总加印数量
2. 点击"+ 添加"按钮添加归属明细行
3. 从下拉菜单选择部门（营报传媒、财经中心、中经未来、产经中心），或选择"其他"并输入自定义名称
4. 输入该部门的加印数量和自留分发数量
5. 快递数自动计算（= 加印数量 - 自留分发）
6. 所有明细行的数量之和应等于临时加印总数
7. 确认报数后，归属明细随报数一起锁定，不可再编辑
8. 归属明细的输入采用防抖保存（1.5 秒无操作后自动保存），避免快速输入时数据被覆盖

**注意**：社用报小计不包含临时加印（临时加印在页面顶部单独显示）。合订本条目已合并计入社用报小计。

**复合项目**：
- **营报传媒**：由 收发室、读者、备用报 三个子项自动合计
- **报社订阅自投/展示**：由 上犹、高铁展示 两个子项自动合计
- 编辑子项数值后，父项合计自动更新

**操作示例**：
1. 在"外埠"输入框中输入：`5575`
2. 在"本市"输入框中输入：`1212`
3. 依次输入其他变动项
4. 查看页面底部的"总印数"，确保数值合理

### 2.3 自动保存

- **实时自动保存**：修改任何数值后，系统会在 1.5 秒后自动保存到服务器
- **保存状态提示**：页面右上角会显示 "⏳ 保存中..." → "✅ 已自动保存"
- **无需手动保存**：数据可以分多天零散填入，随时关闭页面再回来继续编辑
- **保存失败提示**：如遇网络问题，会显示 "❌ 保存失败"，此时请检查网络后刷新页面

### 2.4 确认报数

确认所有数据无误后：
1. 点击"确认报数"按钮
2. 系统会验证：
   - 所有变动项必须有值
   - 数值不能为负
   - 总印数不能为 0
3. 验证通过后，状态变更为"已确认"
4. 确认后，所有数值显示为纯文本（只读），无法再编辑

**作废（管理员）**：
- 如需修改已确认的报数，管理员可以点击"作废"按钮
- 在弹出的对话框中可填写作废原因（可选）
- 作废后，状态重置为"草稿"，可以重新编辑和确认
- 作废操作会记录在修订历史中

**修订历史**：
- 页面底部以时间轴（Timeline）形式展示该期的所有确认和作废记录
- 每条记录包含操作人、操作时间和作废原因（如有）

### 2.5 检查发货明细

1. 点击顶部菜单"发货明细"
2. 在下拉框中选择当期期号（如"2635"）
3. 系统自动显示发货名单，包括：
   - 姓名
   - 电话
   - 地址
   - 数量
4. 检查名单是否正确：
   - 是否包含所有应发送的收件人
   - 数量是否正确
   - 是否有不应发送的收件人

**生成规则**：
- 仅包含 `active` 状态的收件人
- 必须有有效订阅（在起止日期内）
- 频率匹配：
  - **周**：每期都发
  - **半月**：仅期号为偶数时发送
  - **月底**：仅月最后一期发送

### 2.6 手动调整（如需）

#### 修改发货数量
1. 在发货明细表格中找到需要修改的收件人
2. 修改"数量"列的数值
3. 点击"保存修改"按钮

#### 重新生成
如果发货名单完全错误，可以：
1. 点击"重新生成"按钮
2. 系统会删除当前所有发货记录
3. 根据最新的收件人和订阅信息重新计算

### 2.7 导出 Excel

确认发货明细无误后：

#### 导出报数表
1. 点击"导出报数表"按钮
2. 浏览器下载文件：`第2635期报数表.xlsx`
3. 打开 Excel，输入密码：`0611`
4. 检查数据是否正确

#### 导出发货明细
1. 点击"导出发货明细"按钮
2. 浏览器下载文件：`第2635期发货明细.xlsx`
3. 打开 Excel，输入密码：`0611`
4. 检查 7 个 sheets：
   - 对公
   - 读者
   - 样报
   - 周
   - 半月
   - 月底
   - 汇总

#### 导出全部（推荐）
1. 点击"导出全部"按钮
2. 浏览器下载文件：`第2635期全部文件.xlsx`

## 3. 收件人管理

### 3.0 模板管理

模板管理页面用于维护报数项目模板。模板的变更仅影响新创建的期数，已有期数不受影响。

1. 点击左侧菜单"模板管理"
2. 查看当前所有模板项目，支持按类别筛选和排序
3. **新增项目**：点击右上角"新增项目"按钮，填写类别、子类别、显示名称、默认值、是否为变动项、排序
4. **编辑项目**：点击操作列的编辑按钮修改模板
5. **删除项目**：点击操作列的删除按钮，确认后删除

**注意**：删除或修改模板不会影响已创建的期数数据。

### 3.1 新增收件人

1. 进入"收件人管理"页面
2. 点击"新增收件人"按钮
3. 填写表单：
   - **姓名**：必填，如"李四"或"某某公司"
   - **电话**：必填，如"13900139000"
   - **地址**：必填，如"上海市浦东新区某某路123号"
   - **类型**：必选
     - 对公：企业单位
     - 读者：个人订阅
     - 样报：媒体、合作方
   - **频率**：必选
     - 周：每期都发
     - 半月：每两周发一次
     - 月底：月最后一期发
4. 点击"保存"
5. 收件人创建成功，状态自动为"活跃"

**下一步**：为新收件人添加订阅（见 3.4 节）

### 3.2 编辑收件人

1. 在收件人列表中找到需要编辑的收件人
2. 点击"编辑"按钮
3. 修改信息（姓名、电话、地址、类型、频率）
4. 点击"保存"

**注意**：
- 修改类型或频率会影响后续的发货生成
- 已有的发货记录不会自动更新

### 3.3 停发

如果某个收件人不再需要接收报纸（如订阅到期、地址变更未通知等）：

1. 在收件人列表中找到该收件人
2. 点击"停发"按钮
3. 确认操作
4. 状态变更为"停发"
5. 后续生成发货明细时，该收件人不会出现

**停发 vs 删除**：
- **停发**：保留收件人信息和历史订阅记录，可恢复
- **删除**：彻底移除，无法恢复（系统暂不支持删除）

### 3.4 恢复

如果需要重新开始给某个收件人发送报纸：

1. 在收件人列表中筛选"停发"状态
2. 找到需要恢复的收件人
3. 点击"恢复"按钮
4. 确认操作
5. 状态变更为"活跃"

**注意**：
- 恢复后需要检查订阅是否有效
- 如果订阅已过期，需要重新添加订阅

### 3.5 新订

为新收件人添加订阅：

1. 在收件人列表中找到新创建的收件人
2. 点击"订阅"按钮
3. 填写订阅表单：
   - **类型**：选择"新订"
   - **起始日期**：开始发送的日期，如"2026-01-01"
   - **截止日期**：停止发送的日期，如"2026-12-31"
   - **数量**：每期发送的份数，如"2"
4. 点击"保存"

**示例**：
- 张三从 2026 年 1 月 1 日订阅到 2026 年 12 月 31 日，每期 2 份
- 系统会在每次生成发货明细时检查：
  - 当期出版日期是否在 2026-01-01 至 2026-12-31 之间
  - 如果是，生成 2 份的发货记录
  - 如果不是，不生成

### 3.6 续订

为已有订户续订：

1. 在收件人列表中找到需要续订的收件人
2. 点击"订阅"按钮
3. 填写订阅表单：
   - **类型**：选择"续订"
   - **起始日期**：新订阅的开始日期（通常是上次订阅的 `end_date + 1 天`）
   - **截止日期**：新订阅的结束日期
   - **数量**：每期发送的份数（可与之前不同）
4. 点击"保存"

**示例**：
- 张三的订阅到期日期是 2026-12-31
- 续订时填写：
  - 起始日期：2027-01-01
  - 截止日期：2027-12-31
  - 数量：2
- 系统会创建新的订阅记录，类型为"续订"

**续订的好处**：
- 区分新订和续订，便于统计分析
- 可以计算续订次数、续订率等指标
- 了解客户忠诚度

### 3.7 查看订阅历史

1. 在收件人列表中找到某个收件人
2. 点击"订阅历史"按钮（或在详情页中查看）
3. 显示该收件人的所有订阅记录：
   - 类型（新订/续订）
   - 起止日期
   - 数量
   - 创建时间
4. 可以看到该收件人续订了几次，每次订阅多长时间

## 4. 常见问题 FAQ

### Q1: 如何修改已确认的报数？

**A**: 
1. 管理员可以在报数编辑页面点击"作废"按钮
2. 在弹出的对话框中可以填写作废原因（可选）
3. 确认作废后，期数状态重置为"草稿"
4. 然后可以重新编辑数据并再次确认
5. 所有作废记录会保存在修订历史中，可在页面底部查看

### Q2: 收件人过了订阅截止日期会怎样？

**A**: 
- 订阅截止日期过后，该订阅自动失效
- 生成发货明细时，系统会检查 `publish_date` 是否在 `start_date` 和 `end_date` 之间
- 如果不在范围内，不会为该收件人生成发货记录
- 收件人仍然在"收件人管理"中显示，状态仍为"活跃"
- 如需继续发送，请为该收件人添加新的订阅（续订）

### Q3: 休刊期怎么处理？

**A**: 
- 系统已预先配置好 2026 年的休刊日期：
  - 2026-02-16（期号 2641）
  - 2026-02-23（期号 2642）
  - 2026-10-05（期号 2675）
- 在"下一期信息"中，系统会自动跳过休刊日期
- 休刊期不会创建期数
- 休刊期不会生成报数和发货明细
- 期号连续递增，不受休刊影响

### Q4: 半月和月底的发货规则具体是什么？

**A**: 

**半月（Biweekly）**：
- 发送规则：期号为偶数时发送
- 示例：期号 2636、2638、2640... 会发送，2635、2637、2639... 不发送

**月底（Monthly）**：
- 发送规则：月最后一期发送
- 判断方法：比对当期的 `publish_date` 和下一期的 `publish_date`
- 如果两个日期的月份不同，说明当期是本月最后一期
- 示例：如果 2635 期是 1 月 26 日，2636 期是 2 月 2 日，则 2635 期会发送

### Q5: 如何批量导入收件人？

**A**: 
- 目前系统不支持批量导入
- 需要逐个手动添加
- 未来计划增加 Excel 导入功能

**临时方案**：
- 如果收件人数量很多，可以：
  1. 准备好 SQL 插入语句
  2. 直接在数据库中执行
  3. 刷新页面即可看到新增的收件人

### Q6: Excel 密码忘了怎么办？

**A**: 
- 所有 Excel 文件的密码统一为：`0611`
- 如果修改密码，需要在后端代码中修改：
  - 文件：`backend/app/services/excel_service.py`
  - 搜索：`sheet.protection.password = "0611"`
  - 修改为新密码

### Q7: 导出的 Excel 格式不对怎么办？

**A**: 
1. 检查是否存在自定义模板：`backend/app/templates/report_template.xlsx`
2. 如果存在模板，系统会使用模板填充数据
3. 如果不存在，系统会动态创建工作簿
4. 如果格式不满足要求，可以：
   - 创建符合规范的模板文件
   - 或修改代码中的导出逻辑

### Q8: 系统支持多年管理吗？

**A**: 
- 目前仅初始化了 2026 年的刊期表
- 如需增加其他年份：
  1. 参考 `backend/app/seeds/publication_schedule_2026.py`
  2. 创建新的种子文件，如 `publication_schedule_2027.py`
  3. 运行种子数据导入
- 系统架构支持多年，只是需要手动添加刊期数据

### Q9: 发货数量可以为 0 吗？

**A**: 
- 订阅的数量必须大于 0
- 发货记录的数量可以手动调整为 0（表示本期不发）
- 但如果某个收件人长期不发，建议将其状态改为"停发"

### Q10: 如何查看历史期数的数据？

**A**: 
1. 在首页的"期数列表"中可以看到所有已创建的期数
2. 点击某一期可以查看详情
3. 可以查看该期的：
   - 报数数据
   - 发货明细
   - 导出 Excel
4. 已确认或已导出的期数无法修改，但可以查看和重新导出

```bash
cd frontend

```

构建完成后，静态文件会生成在 `frontend/dist` 目录。

```bash
cd backend

uvicorn app.main:app --host 0.0.0.0 --port 8000
```

**注意**：

- 如果存在，会提供静态文件服务
- 访问 `http://your-server:8000` 即可使用系统

### 5.3 使用反向代理（推荐）

**Nginx 配置示例**：

```nginx
server {
    listen 80;
    server_name your-domain.com;

    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
}
```

### 5.4 进程管理（推荐）

**使用 systemd（Linux）**：

创建 `/etc/systemd/system/newspaper.service`：

```ini
[Unit]
Description=Newspaper Report System
After=network.target

[Service]
Type=simple
User=your-user
WorkingDirectory=/path/to/FirstTry/backend
Environment="PATH=/path/to/FirstTry/backend/venv/bin"
ExecStart=/path/to/FirstTry/backend/venv/bin/uvicorn app.main:app --host 0.0.0.0 --port 8000
Restart=always

[Install]
WantedBy=multi-user.target
```

启动服务：
```bash
sudo systemctl enable newspaper
sudo systemctl start newspaper
sudo systemctl status newspaper
```

## 7. 故障排查

### 7.1 后端启动失败

**症状**：运行 `uvicorn app.main:app` 报错

**可能原因**：
1. 虚拟环境未激活
2. 依赖未安装
3. 数据库连接失败
4. `.env` 文件配置错误

**解决方案**：
1. 确保虚拟环境已激活（命令行前有 `(venv)` 标记）
2. 重新安装依赖：`pip install -r requirements.txt`
3. 检查 MySQL 是否运行：`mysql -u root -p`
4. 检查 `.env` 文件中的数据库配置

### 7.2 前端无法访问后端

**症状**：前端页面显示"网络错误"或"无法连接到服务器"

**可能原因**：
1. 后端服务未启动
2. CORS 配置错误
3. 端口冲突

**解决方案**：
1. 确认后端运行在 `http://localhost:8000`
2. 检查 `backend/app/main.py` 中的 CORS 配置
3. 确认前端 API 请求地址正确（`frontend/src/api/` 目录下的文件）

### 7.3 导出 Excel 失败

**症状**：点击"导出"按钮后无响应或报错

**可能原因**：
1. `openpyxl` 库未安装
2. 数据不完整
3. 期数状态不正确

**解决方案**：
1. 确认已安装 openpyxl：`pip show openpyxl`
2. 检查该期的报数数据和发货明细是否完整
3. 确认期数状态为"已确认"或"已导出"

### 7.4 发货明细为空

**症状**：生成的发货明细中没有任何收件人

**可能原因**：
1. 没有活跃的收件人
2. 收件人没有有效订阅
3. 频率不匹配

**解决方案**：
1. 检查收件人列表，确认有"活跃"状态的收件人
2. 检查订阅记录，确认起止日期包含当期出版日期
3. 检查频率规则（半月和月底可能不是每期都发）
