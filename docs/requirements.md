# 需求文档

## 1. 项目背景

《中国经营报》是一份周出版的财经报纸，周五需要生成下周一出版的印数管理表和中通快递发货明细。

### 1.1 业务痛点
- 周手工填写 Excel 报数表，容易出错
- 发货名单管理混乱，订阅状态难以追踪
- 固定数据和变动数据混在一起，效率低下
- 历史数据缺乏系统化管理

### 1.2 解决方案
开发一套 Web 应用，实现：
- 自动生成报数表和发货明细
- 分离固定数据和变动数据
- 管理收件人和订阅信息
- 导出符合规范的 Excel 文件
- 用户认证和权限管理（管理员/操作员）
- 确认锁定与作废审计追踪

## 2. 2026年刊期表

### 2.1 基本信息
- **总周数**：52 周
- **出版期数**：49 期
- **休刊期数**：3 期
- **期号范围**：2635 - 2683

### 2.2 休刊日期
1. **2026年2月16日**：春节休刊（不占用期号）
2. **2026年2月23日**：春节休刊（不占用期号）
3. **2026年10月5日**：国庆节休刊（不占用期号）

### 2.3 出版规律
- 周一出版（除休刊日）
- 周五完成下周的报数和发货准备
- 期号连续递增，休刊周不占用期号

### 2.4 年度刊期表维护
- 管理员可以上传年度刊期 PDF，系统解析出版日期、期号、休刊信息和版数（从"对开XX版"提取）。
- 管理员可以在「刊期表管理」页面按年份查看年度计划摘要、按月份分组的刊期表和上传记录。
- 上传后必须先预览解析结果；只有校验无误的结果才能提交写入正式刊期表。
- 预览行应支持手动新增、删除和修改出版日期/期号/休刊状态，版数可在预览统计区域手动编辑，并在保存前重新校验。
- 非休刊行期号必须连续递增；休刊周不填写期号，也不消耗期号。
- 系统需要保留上传记录、解析摘要、错误信息、上传人和提交时间，便于审计。
- PDF 解析失败、依赖缺失或抽取到无法匹配的日期数字时，系统应给出可理解的错误提示，不应写入正式刊期表或返回未处理的 500。
- 已创建期数受保护：更新年度刊期表时，不得删除已创建期号，也不得修改已创建期号的出版日期。
- 支持文字版 PDF；扫描 PDF 需要先 OCR 成可抽取文本后再上传。

## 3. 周工作流程

### Step 1: 创建新期
- **时间**：周五
- **操作**：点击"创建本期报数"按钮
- **系统行为**：
  - 自动识别下一个期号
  - 自动获取对应的出版日期
  - 优先从上一期复制报数项目；如果是第一期，则从模板初始化
  - 固定项沿用上一期或模板默认值
  - 变动项沿用上一期或模板默认值，等待本期调整
  - 首页仪表盘展示统计卡片（总期数、待处理、本周印数与上周对比、最新报数时间）、一键创建、补录其他期数、报数流程指引、近期印数表格和 ECharts 趋势图

### Step 2: 输入变动数据
- **时间**：周五上午
- **操作**：在报数编辑页面填写橙色高亮的变动项
- **版数**：页面顶部同时显示「计划 XX 版」（来自期刊表）和「实际 XX 版」。创建期次时自动从期刊表获取计划版数作为初始值，确认前可手动修改；不一致时显示警告。版数会影响导出的 Excel 报数表
- **变动项包括**：
  - 北京邮发（外埠）
  - 北京邮发（本市）
  - 北京报零（东部）
  - 北京报零（西部）
  - 广州日报（零售）
  - 广州日报（订阅）
  - 临时加印（支持归属明细分配）
  - 营报传媒
  - 财经中心
  - 备用报

### Step 3: 确认报数
- **操作**：点击"确认报数"按钮
- **实时一致性预警**（确认前即可见）：
  - 报数编辑页与 ZTO-MF 当期 Tab 顶部，只要"中通物流公司"报数合计 ≠ 当期发货明细合计，就显示黄色 ⚠️ Alert 提示差值
  - 草稿阶段也会显示，鼓励用户尽早调整，避免最后一刻才发现对不上
- **系统验证**：
  - 所有变动项必须有值
  - 数值不能为负
  - 总印数不能为 0
  - 报数去向"中通物流公司"合计与当期ZTO-MF份数合计不一致时，确认接口返回核查提示
- **结果**：
  - 状态变更为 `confirmed`
  - 所有数值显示为纯文本（只读），无法再编辑
  - 当期无ZTO-MF时，自动从上一期复制作为本期起始数据，并清空确认状态和发货时间，供人工调整
  - 确认记录保存到修订历史
  - 系统额外保存一条确认快照，记录确认时的报数中通合计、发货明细合计、差值和是否一致
  - 报数编辑页与ZTO-MF页都可查看"确认时快照"与"当前明细状态"，用于识别确认后漂移
  - 上面提到的实时 ⚠️ Alert（草稿期预警）与"确认时快照"（确认后漂移追溯）并存，作用阶段不同

**作废机制**：
- 管理员可以在报数编辑页面点击"作废"按钮撤销确认
- 可填写作废原因（可选）
- 作废后状态重置为 `draft`，可以重新编辑和确认
- 所有确认和作废操作记录在 `report_revisions` 表中，形成完整的审计追踪

### Step 4: 检查发货明细
- **操作**：进入「物流管理 → ZTO-MF」
- **系统行为**：
  - 该页作为当前期中通执行明细的唯一维护入口
  - 用户按期号查看、筛选、维护 `shipping_details`
  - 页面显示当前发货总份数，以及确认时快照/当前状态/是否漂移
- **手动调整**：
  - 可新增、编辑、删除、批量更新ZTO-MF
  - 每次修改都会刷新与报数页对照的当前状态
  - 支持按出刊日期、期号、渠道、子渠道、签约公司、频率、运输方式、状态和姓名筛选

### Step 5: 导出 Excel
- **操作**：点击"导出全部"按钮
- **生成文件**：
  - **报数表**：6 个 sheets（邮发报零订户、赠阅报、备用报、合计、总计、说明）
  - **发货明细**：7 个 sheets（按类型3个 + 按频率3个 + 汇总）
  - **打包文件**：ZIP，内含报数表和发货明细两个 Excel 文件
- **文件命名**：
  - `第2635期报数表.xlsx`
  - `第2635期发货明细.xlsx`
  - `issue_2635.zip`
- **安全性**：所有 Excel 文件密码保护，密码 `0611`
- **追溯**：
  - `/export/report` 会记录 `report_export` 快照
  - `/export/shipping` 会记录 `shipping_export` 快照
  - `/export/all` 会同时记录以上两类快照

### Step 6: 发送给相关方
- **操作**：通过邮件或其他方式发送导出的 Excel
- **接收方**：
  - 印刷厂（报数表）
  - 中通快递（发货明细）
  - 内部存档

## 5. 物流管理

> 左侧导航「物流管理」为展开式子菜单，含 **ZTO-MF**、**收件人**、**邮局投递**（占位 · 功能待接入）三页。

### 5.1 收件人信息
- **姓名**：收件人姓名或单位名称
- **电话**：联系电话
- **地址**：详细地址
- **类型**：对公/读者/样报
- **频率**：周/半月/月底
- **状态**：活跃/停发

### 5.2 三种类型

#### 对公（Business）
- **定义**：企业、机构、单位
- **示例**：某某公司、某某图书馆、某某研究所
- **特点**：通常订阅周期长，数量稳定

#### 读者（Reader）
- **定义**：个人订阅
- **示例**：张三、李四
- **特点**：订阅周期相对短，可能频繁续订或停订

#### 样报（Sample）
- **定义**：媒体、合作方、赠阅对象
- **示例**：某某报社、某某电视台、某某协会
- **特点**：通常不收费，用于宣传或合作

### 5.3 三种频率

#### 周（Weekly）
- **发送规则**：每期都发
- **适用场景**：核心订户、长期合作方

#### 半月（Biweekly）
- **发送规则**：每两周发一次（期号为偶数时发送）
- **适用场景**：预算有限的订户

#### 月底（Monthly）
- **发送规则**：月最后一期发送
- **适用场景**：仅需定期了解内容的订户

### 5.4 状态管理

#### 活跃（Active）
- 正常发送报纸
- 出现在发货明细中

#### 停发（Inactive）
- 暂停发送
- 不出现在发货明细中
- 可随时恢复

### 5.5 操作流程

#### 新增收件人
1. 点击"新增收件人"按钮
2. 填写姓名、电话、地址
3. 选择类型和频率
4. 保存后自动为 `active` 状态
5. 需要再添加订阅才能开始发送

> **地址自动补全**：输入地址时，系统会自动补全缺失的省/市/区前缀（如 `广州市白云区xxx` → `广东省广州市白云区xxx`），支持区县反查市级（如 `顺德陈村xxx` → `广东省佛山市顺德区陈村xxx`）。

#### 编辑收件人
1. 点击收件人列表中的"编辑"按钮
2. 修改信息
3. 保存

#### 停发
1. 点击"停发"按钮
2. 状态变更为 `inactive`
3. 后续不再生成发货记录

#### 恢复
1. 点击"恢复"按钮
2. 状态变更为 `active`
3. 下次生成发货时会包含该收件人

## 5B. ZTO-MF管理

### 5B.1 功能概述
管理中通快递发货的详细明细数据，数据来源为每期的《中通快递发货明细》Excel 文件。明细按期号管理，用户在前端选择 `issue_number` 后查看和维护当期数据，并可看到当期发货总份数。以统一表格+筛选器的方式展示发货记录，支持完整的增删改查操作。

### 5B.2 渠道分类（8种）

| 渠道 | 说明 | 举例 |
|------|------|------|
| 渠道订阅 | 分销商/代理商 | 广州日报（叶剑）、杂志铺（肖波）、国图贸（王程） |
| 对公订阅 | 企业/机构直接订阅 | 北京悦途出行（高铁展示） |
| 个人订阅 | 自有网点的个人读者 | 周读者中的个人订单 |
| 记者站 | 地方记者站内部分发 | 上海站（李广）、广州站（纪玉文）、成都站（党鹏）、西安站（王金龙） |
| 赠阅（监管） | 公家监管部门免费赠送 | 新闻出版署、版本馆等 |
| 赠阅（政府） | 政府单位赠报 | 上犹县各单位 |
| 库房留存 | 库房备用存储 | 马飞（中通老板） |
| 报社留存 | 包车运输到报社 | 程先生 |

### 5B.3 运输方式（4种）
- **中通物流** — 大部分收件人
- **邮政物流** — 上犹等偏远地区
- **包车运输** — 报社留存专用
- **库房留存** — 不发出，存库房

### 5B.4 发送频率
- **周** — 常规每期发送
- **半月** — 隔周发送（含停发状态）
- **月** — 月末统一发送

### 5B.5 签约公司
每条发货明细可关联一个签约公司（`company` 字段），用于在同一渠道内区分不同来源：

| 渠道 | 签约公司 | 说明 |
|------|---------|------|
| 对公订阅 | 北京悦途出行 | 高铁展示 VIP 厅 |
| 渠道订阅 | 广州日报 | 叶剑 |
| 渠道订阅 | 成都杂志铺 | 肖波 |
| 渠道订阅 | 国图贸 | 月底整月发 |
| 记者站 | 上海站 | 李广 |
| 记者站 | 广州站 | 纪玉文 |
| 记者站 | 成都站 | 党鹏 |
| 记者站 | 西安站 | 王金龙 |

前端支持按签约公司多选筛选，可选一个或多个公司查看对应数据。

### 5B.6 数据用途
- 物流费用结算：按运输方式（中通物流）筛选，统计可计费发货量
- 渠道分析：按渠道分类汇总各类订阅数据
- 发货状态追踪：正常/停发状态管理
- 期次复用：确认报数时可从上一期复制明细，避免重复录入稳定收件名单

## 6. 订阅管理

### 6.1 订阅类型

#### 新订（New）
- 首次订阅
- 记录新增订户

#### 续订（Renewal）
- 再次订阅
- 用于统计续订率、客户忠诚度

### 6.2 订阅属性
- **起始日期**：开始发送日期
- **截止日期**：停止发送日期
- **数量**：每期发送份数

### 6.3 有效性判断
订阅在以下条件下有效：
```
start_date <= publish_date <= end_date
```

超过 `end_date` 后，该订阅失效，不再生成发货记录。

### 6.4 数据分析支持
通过订阅记录可以分析：
- **续订次数**：某个收件人续订了几次
- **续订率**：多少订户选择续订
- **订阅周期**：平均订阅时长
- **客户价值**：累计订阅份数和时长

### 6.5 操作流程

#### 为新收件人添加订阅
1. 创建收件人后，点击"订阅"按钮
2. 选择类型："新订"
3. 填写起止日期和数量
4. 保存

#### 为已有收件人续订
1. 在收件人列表中点击"订阅"按钮
2. 选择类型："续订"
3. 填写新的起止日期（通常从旧订阅结束后开始）
4. 保存

## 7. 验证规则

### 7.1 报数数据验证

#### 创建新期时
- 期号必须唯一
- 期号必须来自刊期表
- 出版日期必须匹配刊期表

#### 编辑报数时
- 所有变动项不能为空
- 数值不能为负
- 已确认的期数不能修改

#### 确认报数时
- 所有变动项必须有值
- 总印数不能为 0
- 状态必须为 `draft`
- 报数去向为"中通物流公司"的合计应与当期ZTO-MF份数合计一致；不一致时允许确认，但必须提示用户核查
- 如果当期没有ZTO-MF，系统应从上一期复制一份作为本期起始数据，便于人工调整

### 7.2 收件人验证

#### 创建/编辑时
- 姓名不能为空
- 电话不能为空
- 地址不能为空
- 类型必须为：business/reader/sample
- 频率必须为：weekly/biweekly/monthly

### 7.3 订阅验证

#### 创建时
- 起始日期不能晚于截止日期
- 数量必须大于 0
- 类型必须为：new/renewal

### 7.4 发货验证

#### 生成时
- 收件人必须为 `active`
- 必须有有效订阅
- 频率必须匹配当期

## 8. Excel 文件规范

### 8.1 报数表

**文件名**：`第{期号}期报数表.xlsx`

**Sheet 结构**：
1. **邮发、报零、订户**：主要报数数据
2. **赠阅报**：免费赠送的报纸
3. **备用报**：备用库存
4. **合计**：各类别小计
5. **总计**：总印数

**数据来源**：`report_entries` 表

## 9. 业务规则总结

### 9.1 期数管理
- 期号连续递增
- 休刊期不创建期数
- 每期状态：draft → confirmed → exported
- 确认后不可修改（除非手动改数据库）

### 9.2 报数逻辑
- 固定项自动填入
- 变动项必须手动输入
- 每条报数项都有去向：北京邮发→北京市报刊发行局，北京报零→北京市报刊零售公司，合订本→印厂，其余类别→中通物流公司
- 总印数 = 所有项之和（不含临时加印_自留等子分配项）
- 确认时比较"中通物流公司"报数合计与当期ZTO-MF合计，不一致时给出核查提示
- 确认前可多次保存草稿

### 9.3 发货逻辑
- 休刊期不生成发货记录
- 仅为 `active` 状态的收件人生成
- 必须有有效订阅（在起止日期内）
- 频率必须匹配当期
- 数量来自订阅记录

### 9.4 订阅逻辑
- 新订和续订分别记录
- 订阅可重叠（系统取最新有效订阅）
- 过期订阅自动失效
- 支持同一收件人多次续订

### 9.5 Excel 导出
- 使用模板（如存在）或动态生成
- 所有文件密码保护
- 文件名包含期号
- 支持单独导出或合并导出
